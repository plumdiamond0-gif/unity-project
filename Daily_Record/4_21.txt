원래는 WeaponPrefabTable에서 bool 형으로 canCharge, canSlow, canStun...을 선언해서 if 문으로 이펙트를 각 무기마다 적용시키려 했음

문제 : 그러나 if문이 너무 반복되고 보기에 혼잡함.

그래서 각 이펙트 ScriptableObject 스크립트들에 클래스를 따로 만들고 IWeaponEffect라는 인터페이스에 구현하게 만듦.

그리고 WeaponPrefabTable에 적용할 이펙트들을 List<ScriptableObject>로 저장함

PlayerAttack에서 GameManager를 통해 CannonBall을 만들고 거기서 GetComponent를 통해 클래스 가져옴.

무기 데이터를 탄환에 주입해 탄환이 독립적으로 처리하게 만듦.
매 프레임마다 넘겨주는 게 아닌 발사 시에만 한 번씩 넘겨줌.

    foreach (var effectobjs in data.effects)
//현재 무기 데이터 내의 이펙트들을 돌며 반복문 실행
    {
        if(effectobjs is IWeaponEffect effect)
//ScriptableObject로 저장된 객체가 IWeaponEffect를 구현하고 있다면 해당 객체를 IWeaponEffect 타입으로 참조하여 effect로 사용
        {
            effect.Apply(target);
//각 이펙트들은 인터페이스의 Apply 함수를 구현하고 있어 차례대로 이펙트들의 함수 호출
        }
    }



구현과 상속의 차이

상속 = “이미 있는 기능을 물려받고 확장”
부모 클래스가 코드(기능)를 가지고 있고 자식이 그걸 그대로 쓰거나 수정(override) 가능

구현(인터페이스) = “이 규칙을 반드시 지켜라”
함수 틀만 있고 (내용 없음) 반드시 직접 구현해야 함
