무기를 업그레이드한 결과 증가된 데미지량,공격속도량도 
So원본 데이터 값을 바꾸는 게 아니라 세이브매니저에 저장해야 할지 :

SO 직접 수정 -> 
1. 모든 플레이어에게 적용됨
SO는 원본 데이터라서
weaponData.baseDamage += 10;
해버리면 그 SO를 사용하는 모든 무기가 영향을 받아.

2. 저장/불러오기 꼬임
게임 껐다 켜면
SO는 원래 값으로 돌아가는데
업그레이드 내용은 사라질 수 있음.

3. 밸런스 수정이 어려움
나중에
기본 공격력 20 → 30
으로 패치하고 싶을 때
SO는 기본값만 관리하고
세이브 데이터는 업그레이드량만 관리해야 계산이 깔끔함.

각 무기마다 레벨만 저장해 놓고 레벨에 따라 증가하는 데미지량은 다른 곳에서 처리할지
아니면 그냥 SD_User에 증가한 값도 다 넣을지 :

-무기 레벨에 따른 성장량이 고정이라면 ->
public Dictionary<WeaponState, int> weaponLevel = new()
{
    {WeaponState.Base, 0},
    {WeaponState.Improved, 0},
    ...
};
이것만 저장하고,
실제 데미지는
int level = SaveManager.CurrentData.weaponLevel[WeaponState.Fire];
int finalDamage =
    weaponSO.baseDamage +
    level * weaponSO.damagePerLevel;
이런 식으로 계산.

레벨과 증가량을 둘 다 저장하면 중복 데이터가 됨

1.
레벨 = 5
증가 데미지 = 40
를 저장했는데
나중에 밸런스 패치로
레벨당 +8 → +10
으로 바꾸면
저장된 증가데미지 40과 계산 결과 50이 충돌함.

2.
저장 데이터가 작아짐
지금
Fire = 7
한 줄이면 끝인데
FireLevel = 7
FireDamageBonus = 35
FireAttackSpeedBonus = 0.14f
...
이런 식으로 늘어나기 시작함.
계산식 수정이 쉬움

3.
1레벨 : +5
2레벨 : +12
3레벨 : +20
같이 성장 공식을 바꿔도
저장된 레벨만 있으면 자동 적용됨.


증가량 자체를 저장 ->
예를 들어 로그라이크 리워드에서
공격력 +10%
공격력 +15
공속 +0.2
같이 플레이어가 임의로 선택한 영구 강화라면
그건 레벨만으로 복원할 수 없으니까
public class SD_Weapon
{
    public int level;

    public float damageBonus;
    public float attackSpeedBonus;
    public float critBonus;
}
처럼 저장.
